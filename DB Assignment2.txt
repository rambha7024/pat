🧱 TABLE 1: DEPARTMENTS
CREATE TABLE departments (
    dept_id     NUMBER PRIMARY KEY,
    dept_name   VARCHAR2(50) UNIQUE NOT NULL,
    location    VARCHAR2(100),
    manager_id  NUMBER,
    budget      NUMBER(12,2)
);

🧱 TABLE 2: EMPLOYEES
CREATE TABLE employees (
    emp_id        NUMBER PRIMARY KEY,
    emp_name      VARCHAR2(100) NOT NULL,
    email         VARCHAR2(200) UNIQUE,
    phone_number  VARCHAR2(15) NOT NULL,
    salary        NUMBER(10,2) CHECK (salary > 0),
    commission    NUMBER(10,2),
    dept_id       NUMBER,
    manager_id    NUMBER,
    hire_date     DATE NOT NULL,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
);


⚠️ Intentional design choices

manager_id is self-referencing → enables self-join questions

commission nullable → NVL / CASE / filters

Multiple employees per department → GROUP BY / HAVING

Some departments can have no employees
